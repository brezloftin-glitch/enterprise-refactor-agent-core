# 🚀 Enterprise Legacy Refactor Agent (Core System)

> **⚠️ 访问声明 (NDA Notice):** > 本项目属于企业内部核心链路演进的底层自动化工具。受限于严格的商业代码保密协议，本 Public 仓库仅用于展示系统整体的架构设计蓝图、多智能体工作流以及技术选型。核心的 AST 解析脚本、业务约束插件及测试沙箱代码暂不开源。

## 📖 项目全景与底层架构选型

本系统的初衷是解决企业级历史包袱代码（Legacy Code）在微服务演进过程中的自动化重构难题。为了构建高并发、高容错的 Agent 协同网络，本项目的底层通信与基础编排逻辑，深度参考并集成自业界领先的开源框架 **[微软 AutoGen](https://github.com/microsoft/autogen)**。

在此，向开源社区提供的卓越多智能体底座表示敬意。通过引入 AutoGen 的多角色对话博弈机制，我们初步实现了“感知-决策-执行-验证”的闭环流。

## ⚙️ 私有化深度定制层 (Custom Enhancements)

考虑到企业核心交易链路重构的特殊性与极高的容错要求，我们没有直接使用原生的通用能力，而是基于开源底座进行了深度的场景化定制开发，主要包含以下三大增强核心：

### 1. 定制化全景感知层 (AST Perception)
重写了原生的简单文件读取模块。接入了专为老旧 Java/Spring 体系定制的抽象语法树（AST）解析插件，能够精准提取跨文件的全局类图与高密度调用链，将非结构化代码转化为大模型更易理解的结构化图谱（JSON）。

### 2. 业务级强约束与自愈层 (Validation & Self-Healing)
摒弃了原生框架中过于宽泛自由的对话逻辑。我们在 Execution Agent（执行器）和 Validation Agent（验证器）之间植入了严格的**“沙箱自愈熔断机制”**：
- 强制要求 LLM 生成配套的 Mockito/JUnit 单元测试。
- 自动拉起本地 Maven 沙箱进行编译校验。
- 精准提取 Error Stack Trace，驱动上游 Agent 进行最高 3 次的闭环“自愈纠错”。

### 3. 百万级上下文模型适配层 (MiMo LLM Adaptation)
大规模工程的跨文件重构极度依赖全局上下文，传统模型 128K 的窗口在引入依赖图谱后极易发生“截断幻觉”。目前我们正在重构底层的模型调度适配层，全面对接具备百万级超长上下文能力的 **MiMo-V2.5-Pro** 模型，以实现真正的无分块（Zero-Chunking）全局架构演进。

## 📊 内部压测指标 (Alpha Phase)
在近期的边缘业务线压测中，本系统取得了以下阶段性成果：
- **研发人效:** 跨模块复杂调用链路梳理耗时下降 **60%**。
- **自愈质量:** 依靠沙箱熔断机制，机器重构代码的初次编译通过率稳步提升至 **82%**。
- **算力吞吐:** 在高频纠错循环下，单日峰值 Token 消耗已突破 **8M+**（强烈依赖高额度与高并发支持）。
